# 系统使用说明

## 一、启动系统

```bash
cd risk-monitor-system
pip install -r requirements.txt     # 首次运行需安装依赖
python data/seed_data.py            # 首次运行需初始化数据库
python run.py                       # 启动Web服务
```

访问 **http://127.0.0.1:5000** 进入系统。

---

## 二、五模块操作指南

顶部导航栏包含 **企业选择器**（可切换企业）和 **5个模块标签页**。

### M1 数据接入 —— 录入企业公开信息

| 操作 | 步骤 |
|------|------|
| **添加企业** | 点击「添加企业」→ 填写企业名称、股票代码、行业 → 在文本框粘贴公开信息 → 点击「创建并分析」 |
| **补充数据** | 在文本框中粘贴更多信息 → 点击「录入文本数据」→ 系统自动重新分析 |
| **上传文件** | 点击「选择文件」→ 选择 DOCX/Excel/TXT 文件 → 系统自动提取文本并分析 |

> **录入建议**：粘贴企业年报「风险因素」章节、行业分析报告摘要、以及ESG报告中涉及风险的内容。

---

### M2 风险分析 —— 查看 AI 自动分析结果

进入该页面即可看到：

- **风险清单**：按得分降序排列的完整风险列表（编号/名称/类别/P/I/得分/等级/趋势/应对策略）
- **风险矩阵**：5×5 彩色热力图（×1~×25），颜色越红风险越高
- **预警列表**：红/橙/黄三级预警
- **统计概览**：重大/高度关注/一般风险数量、平均得分

点击「运行完整分析」可重新执行 M1→M2→M3→M4→M5 全流程。

---

### M3 内控机制 —— 查看自动生成的内控评估

四个可展开的子模块：

| 子模块 | 内容 |
|--------|------|
| **6.1 治理结构** | 行业适配的治理模式、董事会架构、专门委员会建议、独董占比评估 |
| **6.2 三道防线** | 业务部门→风险管理部→内部审计，含每道防线的针对性职责和改进建议 |
| **6.3 内控制度** | 每项制度可展开查看：覆盖了哪些风险、有效性评估、具体改进措施 |
| **6.4 覆盖评估** | 内控覆盖率仪表盘、未覆盖风险列表、优先级改进建议 |

所有内容根据**录入的企业数据和风险分析结果**自动生成，不同企业、不同行业的结果完全不同。

---

### M4 监督改进 —— 查看自动生成的监督体系

四个可展开的子模块：

| 子模块 | 内容 |
|--------|------|
| **7.1 监督机制** | 董事会/内部审计/外部审计/监管监督等机制评估，频次根据风险水平自动调整 |
| **7.2 信息披露** | 各类披露（年报/ESG/季报/临时公告）的风险披露状态和改进建议 |
| **7.3 问题与改进** | 自动识别的问题（重大风险应对、内控覆盖不足、趋势上升等），每项可展开查看详情 |
| **7.4 PDCA循环** | 计划→执行→检查→改进四阶段，含该企业风险数据的针对性内容 |

---

### M5 报告输出 —— 生成并下载报告

两种报告格式：

| 格式 | 内容 |
|------|------|
| **Word完整报告** | 6节完整报告（摘要→公司概况→风险评估→风险应对→内控机制→监督改进），匹配主报告结构 |
| **Excel风险清单** | 风险编号/名称/类别/P/I/得分/等级/趋势/应对策略，带颜色标记 |

点击按钮后系统自动生成，弹出下载链接。

---

## 三、企业切换

顶部导航栏的企业选择下拉框中可切换已录入的所有企业。切换后，M2~M5 五个模块的数据自动切换对应该企业的分析结果。

---

## 四、支持五类行业

系统内置了 5 个行业的分析模板，M3/M4 会根据行业自动匹配：

| 行业 | 适配特点 |
|------|----------|
| 互联网科技 | AI治理、数据安全、反垄断合规 |
| 制造业 | 安全生产、供应链风险、质量管理 |
| 金融 | 信用/市场/流动性风险、资本充足率、AML |
| 消费 | 食品安全、品牌声誉、渠道管理 |
| 能源 | HSE管理、碳排放、能源价格对冲 |

---

## 五、数据存储

所有录入的企业数据和分析结果存储在 `risk-monitor-system/data/risk_monitor.db`（SQLite 数据库）中，删除该文件后重新运行 `seed_data.py` 即可恢复初始状态。
